# Быстрый старт с агентской инфраструктурой

## Кратко: что изменилось

Репозиторий теперь поддерживает единый workflow для всех агентов (Windsurf, Pi, Codex, Claude). Основные компоненты:

- `AGENTS.md` — правила проекта (читается всеми агентами автоматически)
- `.agents/skills/` — набор навыков для выполнения задач
- `docs/tasks/` — хранилище задач

---

## Как создать задачу (шаг за шагом)

### 1. Создай спецификацию задачи

Агент знает навык `spec-writer`. Просто попроси:

```
Вызови spec-writer и создай задачу на <описание>
```

Или создай файл вручную по шаблону:
`docs/tasks/YYYY-MM-DD--краткое-название.md`

Шаблон — в `.agents/skills/spec-writer/assets/spec-template.md`

### 2. Построй план через planning_supervisor

```
Вызови planning_supervisor для задачи docs/tasks/YYYY-MM-DD--название.md
```

Супервизор запустит цикл: план → критика → доработка → верификация.

### 3. Выполни план

Агент получит готовый plan.md и выполнит шаги.

### 4. Проверь результат через qa-gate

```
Вызови qa-gate для проверки задачи docs/tasks/YYYY-MM-DD--название.md
```

Скрипт `./.agents/skills/qa-gate/scripts/verify.sh` проверит:
- PHP синтаксис
- Линтер
- Тесты
- Соответствие спецификации

---

## Доступные навыки (skills)

| Навык | Когда использовать |
|-------|-------------------|
| `spec-writer` | Создать новую задачу или обновить спецификацию |
| `planning` | Построить план выполнения (обычно вызывается супервизором) |
| `plan_critic` | Проверить план на ошибки и противоречия |
| `plan_verifier` | Формальная проверка плана на соответствие правилам |
| `planning_supervisor` | Полный цикл планирования с критикой и верификацией |
| `qa-gate` | Финальная проверка выполненной задачи |

## Примеры команд

### Создать задачу на рефакторинг
```
Вызови spec-writer и создай задачу на рефакторинг 
WP_Field::render() с целью разделения ответственности
```

### Запустить полный цикл планирования
```
Вызови planning_supervisor для задачи 
docs/tasks/2025-01-15--refactor-render.md
```

### Проверить выполненную работу
```
Вызови qa-gate для задачи docs/tasks/2025-01-15--refactor-render.md
```

## Файловая структура задач

```
docs/tasks/
├── README.md              # Этот гайд
├── 2025-01-15--task-a.md  # Задача A
└── 2025-01-20--task-b.md  # Задача B
```

Имя файла: `ГГГГ-ММ-ДД--краткое-описание.md`

---

## Что делать если что-то не работает

1. **Агент не находит навык** — проверь что путь правильный: `.agents/skills/<name>/SKILL.md`
2. **verify.sh падает** — запусти вручную: `bash ./.agents/skills/qa-gate/scripts/verify.sh`
3. **Нужно больше контекста** — агент автоматически читает `AGENTS.md` в начале сессии

---

## Минимальный рабочий процесс (копируй-вставляй)

```
1. Вызови spec-writer для задачи: "<описание>"
2. Вызови planning_supervisor для созданной задачи
3. Выполни полученный план
4. Вызови qa-gate для проверки
```

Всё. Никакой магии — просто структурированный процесс.
