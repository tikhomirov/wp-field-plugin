Сделай сессию по шагам:
1) Перед изменениями прочитай:
   - `AGENTS.md`
   - `tree.md`
   - `API.md`
   - `docs/wp-field/analysis.md`
   - `docs/wp-field/plan.md`
   - `docs/wp-field/decision-log.md`
2) Сверь задачу с приоритетами из `docs/wp-field/plan.md`.
3) Дай короткий план действий.
4) Внеси минимальные правки без поломки legacy API.
5) Запусти только релевантные проверки (не весь набор без необходимости).
6) В конце обнови минимум один контекстный файл: `analysis.md`, `plan.md`, `decision-log.md` или `AGENTS.md`.
7) В ответе покажи:
   - список изменённых файлов,
   - какие команды запускались,
   - что рекомендовано следующим шагом.
